SpringBoot 配置文件

Spring Boot使用一个全局的配置文件

- application.properties
- application.yml
  • 配置文件放在src/main/resources目录或者类路径/config下
  • .yml是YAML(YUML Ain`t Markup Language)语言的文件，以数据为中心，比json,xml等更适合做配置文件
  • 注意：.yml文件，以数据为中心
  • 全局配置文件的可以对一些默认的值进行修改

    作用

  • 配置文件的作用：修改SpringBoot自动配置的默认值；
    SpringBoot在底层都给我们配置好；

    .yml语法

    基础语法

  • 键值对： K:(空格)V 空格必须有
  • 已空格的缩进来控制层级关系；
    • 只要左对齐的一列数据，都是同一个级别的数据
    • 属性和值大小写敏感
  • 值的写法
    字面量：普通的值（数字，字符串，布尔）
    k：v v直接来写即可
    字符串默认不用加单引号或者双引号
    注意：单引号会转移特殊字符
  • 对象,map(属性和值)；
    对象还是k：v的方式
  • 行内写法
  • 数组（list，set）:
    用- 值表示数组中的一个元素

    .properties文件配置

    @Value(“”)注解

    @Value获取数据与@ConfigurationProperties获取比较

对比 @ConfigurationProperties @Value
功能 批量注入配置文件中的属性 单一指定
松散绑定 支持lastName对应last-name 不支持
SpEl 不支持 支持
JSPR303数据效验 支持如@Email，@NotNull 不支持
复杂类型数据(map) 支持 不支持(基本数据类型)
  • 注意：在业务逻辑中获取配置文件中某项值，使用@Value
    JavaBean和配置文件映射使用@ConfigurationProperties

@PropertySource(只能用于perperties文件)和@ImportResource读取外部配置文件

  • @PropertySource 加载指定路径下的文件

  • @ImportResource: 导入Spring的配置文件，让Spring的配置文件生效

  • Spring 配置文件

  • 注意：在启动类上加入@ImportResource(locations = {“classpath:beans.xml”})注解

    SpringBoot推荐该容器中添加组建的方式：全注解

    ${random.int(10)},${random.int[1024,65536]}

  • 属性配置占位符

  • 可以在配置文件中应用之前那个配置过的属性(优先级前面配置过的这里都能使用)。
    ${app.name:默认值}来指定找不到属性时的默认值

  • Run–》Edit–>program arguments: –spring.profiles.active=dev
    打包 执行 Java -jar spring-boot-02-config-0.0.1-SNAPSHOT.jar –spring.profiles.active=dev;
  • 虚拟机参数：
    -Dspring.properties.active=dev4m

配置文件加载位置

  • springBoot启动会扫描已下位置的application.properties/.yml文件作为Springboot的默认文件
  • file: ./config/
  • file: ./
  • classpath:/config/
  • classpath：/
  • 注意：优先级由高到低，高优先级配置会覆盖低优先级的配置
  • SpringBoot会从这四个位置全部加载主配置文件：互补配置
    我们还可以通过spring.config.location=G:/application.properties来改变默认的配置文件位置（外部部配置加载）
  • 项目打包后，我们可使用命令行参数的模式，启动项目的时候来指定配置文件的新位置；指定配置文件和默认加载的这些配置文件共同起作用的形式互补配置；

    SpringBoot自动配置原理

  • 配置文件能写什么？怎么写？自动配原理？
    自动配置原理：
  1. Spring启动的时候加载主配置类，开启自动配置功能@EnableAutoConfiguration
  2. @EnableAutoConfiguration的作用：
    • 利用EnableAutoConfigurationImportSelector该容器中导入一些组件？
    • 可以查看selectImports()方法的内容：
    • List configurations = getCandidateConfigurations(annotationMetadata,attributes);
      获取候选的配置
      • SpringFactoriesLoader.loadFactorynames()扫描所有jar包下类路径的META-INF/Spring.factories
      • 把扫描到的文件内容包装成properties对象
      • 从properties文件中获取到EnableAutoConfiguration.class类(类名)对应的值然后把它们添加到容器中；
      • 将类路径下META-INF/Spring.factories里面的配置的所用EnableAutoConfiguration的值加入到容器中；
  • META-INF/Spring.factories配置文件(位置：spring-boot-autocongfigure)
  • 注意：上面每一个这样的 xxxAutoConfiguration类都是容器中的一个组件，都加入到容器中；用他们做自动配置；
  1. 每一个自动配置类进行自动配置功能：
    • 已HttpEncodingAutoConfiguration(Http编码自动配置
      )为例解释自动配置原理；
  • 注意：根据当前文件中不同条件判断，决定这个配置是否生效？
    一旦这个配置内生效：这个配置类就会该容器中添加各种组件：这些组件的属性是从对应的properties类中获取的，这些组件中的属性是从对应的properties类中获取的，这些类里面的每一个属性又是和properties文件绑定的
  • 配置properties

    SpringBoot的精髓：

  1. SpringBoot启动会加载大量的自动配置类
  2. 我们要看到我们需要的功能有没有SpringBoot默认配置写好的自动配置类，
  3. 这个自动配置类中到底有哪些组件;(如果我们配置的组件有，我们就不需要配置)
  4. 给容器中自动配置添加组件的时候，会从properties类中获取某些属性，我们就可以在配置文件中指定这些属性的的值；
  • xxxAutoConfiguration:自动配置类；
    给容器中添加组件
    xxxProperties:封装配置文件中相关属性；

    IDEA快捷键，快捷搜索ctrl+N（如输入”*AutoConfiguration”）
