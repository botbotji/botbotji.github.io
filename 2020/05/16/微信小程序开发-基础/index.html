    • 提供两种形式：
      1. 字符串，代表在for循环的array中变量的某个属性，该属性的值需要是列表中唯一的字符串或数字，且不能动态改变。
      2. 保留关键字this代表在for循环中的item本身是唯一的字符串或者数字，
        如：当数据改变触发渲染层重新渲染的时候，会校正带有 key 的组件，框架会确保他们被重新排序，而不是重新创建，以确保使组件保持自身的状态，并且提高列表渲染时的效率。

小程序的MVVM架构

  • 小程序MVVM：
    1. 逻辑层(AppService,–js): 内部提供MINA框架实现绑定
    2. 视图层(View – wxml，wxss)
  • Vue(声明式编程)的MVVM： M->Model(数据),V->View,VM->ViewModel
    1. 将model的数据通过ViewModel中DataBinding绑定到view
    2. View通过事件监听——》DOM listeners

配置小程序

  • 小程序把很多开发需求被规定在了配置文件中。

  • 常见配置文件

    1. project.config.json:项目配置文件，比如项目名称，appid等，比如别人导入会优先读取这个文件生成相关配置，如版本等;

    2. sitemap.json:小程序搜索相关

      • 补充知识点：
        sitemap 的索引提示是默认开启的，如需要关闭 sitemap 的索引提示，可在小程序项目配置文件 //project.config.json 的 setting 中配置字段 checkSiteMap 为 false

    3. app.json:全局配置

      • page string[] 必填 页面路径列表
        小程序所有的页面必须都在pages中进行注册，其中每一项都对应一个页面的路径信息
      • window Object NO 全局默认的窗口表现
      • tabBar Object 否 底部tab栏的表现

    4. page.json:页面配置

      • 配置单个页面的表现
      • 覆盖全局配置

        小程序的双线程模型

      • 页面渲染的过程
      1. 初始化渲染：WXML可以先转成JS对象，在渲染出真正的DOM树
      2. 数据发生变化：(不会刷新整个页面)
        通过setData把msg数据从”Hello World”变成”Goodbye”
        产生的JS对象对应的节点就会发生变化，
        此时可以(通过diff算法)对比前后两个JS对象得到的变化的部分，
        然后把这个差异应用到原来的DOM树上，
        从而达到更新UI的目的，这就是”数据驱动”的原理。

        小程序启动流程

        (当你第一次点击某个小程序)
        下载小程序 –>启动小程序–>加载解析app.json –> 注册App()– 执行App的声明周期
        通过注册App() –>加载自定义组件代码注册自定义组件 –>加载解析page.json
        -->渲染层加载渲染page.wxml
-->逻辑层注册Page() -->执行Page生命周期
