  • 事件捕获是从外向内，事件冒泡是从内向外

    自定义组件

  • 命名：标签名只能是小写字母，中划线，下划线的组合，所以自定义组件的标签名也只能包含这些字符

  • 在根目录下创建一个目录在其中创建一个页面，在这其中写这个组件

  • 导入就是在你想用到的页面的json文件中导入

  • 然后直接就可以使用自定义标签

    注意：

    • 自定义组件也可以引入自定义组件，引入方法类似于页面引入自定义组件的方式
    • 全局注册app.json中”usingComponents”: {“my-cpn”:”/cpn/my-cpn/my-cpn”}，即可
    • 组件内的class样式，只能对组件Wxml内的节点生效，对于引用组件的page页面不会生效；
    • 组件内不能使用id选择器，属性选择器，标签选择器
    • 外部使用标签选择器，会对组件内产生影响。

      如何让class可以相互影响

  • 在Component对象中，可以传入一个options属性，options其中有一个stylesolation(隔离)属性。

  • stylesolation有三个取值：

    • isolated表示启用样式隔离。在自定义组件内外，使用class指定的样式将不会相互影响(默认取值);

    • apply-shared表示页面wxss样式将影响到自定义组件，但自定义组件wxss中指定的样式不会影响页面;

    • shared表示页面wxss样式将影响到自定义组件，自定义组件wxss中的指定的样式也会影响页面和其他设置

      组件间通信

    • 组件间的基本通信方式有以下几种。

    • WXML 数据绑定：用于父组件向子组件的指定属性设置数据，仅能设置 JSON 兼容数据（自基础库版本 2.0.9 开始，还可以在数据中包含函数）。具体在 组件模板和样式 章节中介绍。

      • 数据绑定：在js文件中data域中设置一些初始化数据，后者空的数组等类型的数据，然后通过事件，生命周期中event对象获取数据，然后通过this.setData:()=>{this.data.name = this.event.currentTarget.dataset.name} 这样的方式赋值，进行数据交互

    • 事件：用于子组件向父组件传递数据，可以传递任意数据。
      ？？？？？？？？？？？？？？？？？？？？？？
      如果以上两种方式不足以满足需要，父组件还可以通过 this.selectComponent 方法获取子组件实例对象，这样就可以直接访问组件的任意数据和方法。

      组件的生命周期

  • created ：组建的实例被创建完成时调用，注意，这个生命周期只应该用于给组件 this 添加一些自定义属性字段，不能调用setData方法设置数据

  • attached ：组件初始化完成，进入页面节点数时被调用，“绝大多数初始化工作可以在这个时机进行”

  • datached ：离开这个节点数时触发，注意，页面退出，组件还在这个节点树，会触发
