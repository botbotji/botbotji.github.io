javaScript 模块 ==》运行时,文法和执行过程

  • 运行时类型: 代码实际执行过程中我们用到的类型。
  • 所有的类型数据都会属于 7 种语言类型之一。
    • 语言类型==》广泛用于变量,函数参数，表达式，函数返回值等
  • 从变量，参数，返回值等表达式中间结果，任何 Javascript 代码运行过程中产生的数据，都具有运行时类型。

Undefined 未定义

  • 任何变量赋值之前是 Undefined 类型，值为 undefined，一般我们用全局变量 undefined 来表达这个值，或者 void 运算来把任意一个表达式变成 undefined
  • javascript 代码中，undefined 是一个变量，而不是一个关键字
  • 由于是变量，为了避免无意中被篡改，建议使用 void 0 来获取 undefined 值
  • undefined 跟 Null 有一定的表意差别，Null 表示的是：”定义了但是为空”
  • 在实际编程中，我们一般不会把变量赋值为 undefined，这样可以保证所有的值为 undefined 的变量，都是从未赋值的自然状态

String

  • String 表示的是文本数据。String 有最大长度是 2 的 53 次方-1
  • 注意：这个最大长度，并不完全是你理解的字符数
  • 因为 String 的意义并非’字符串’，而是字符串的 UTF-16 编码,其操作的方法都是针对 UTF-16 的编码，所以，字符串的最大长度，实际上受字符串编码长度的影响
  • 特点：无法变更，String 具有值类型的特征

Number

  • number 基本符合双精度浮点数的规范
  • 额外的语言场景下，规定了几中例外的情况：
    1. NaN 不是数字
    2. Infinaty 无穷大(不让除以 0 出错，而引入了无穷大的概念)
    3. -Infinaty 负无穷大
    4. 注意：javaScript 中有+0 和-0，在加法类运算中他们没有区别，但是在除法的场合则需要特别留意区分，”忘记检测除以-0，而得到负无穷大”的情况会经常会导致错误，而区分+0 和-0 的方式，正式检测 1/x 是 Infinaity 还是-Infinity
    • 根据浮点数的定义，非整数的 Number 类型无法用==(=== 也不行)来比较
    • 0.1+0.2 不能= 0.3 这是由于浮点数运算的精度问题导致等式左右的结果并不是严格的相等，而是相差了微小的值
      • 针对情况,0.1+0.2==0.3 返回 false，这里的错误不是结论，而是比较的方法，正确的比较方法是用 javaScript 提供的最小精度值：
        Math.abs(0.1+0.2-0.3) <= Number.EPSILON
      • 说明：检查等式左右两边差的绝对值是否小于最小精度，才是正确的比较浮点数的方法。结果 true

Symbol 符号

  • 一切非字符串的对象 key 的集合，在 ES6 中对，整个对象系统被用 Symbol 重塑

  • Symbol 可以具有字符串的类型的描述，但是即使描述相同，symbol 也不相等

  • 创建及使用

    • 创建 Symbol 的方式是使用全局的 Symbol 函数。
      var mySymbol = Symbol(“my symbol”)

Object

  • 为什么给对象添加的方法==》能使用到基本数据类型上？？？？

  • 在 javascript 中对象的定义是“属性的集合”。属性分数据属性和访问器属性，二者都是 key-value 结构，key 可以是 String 或者 Symbol 类型

  • 注意：javascript 中的”类”仅仅是运行时对象的一个私有属性，而 JavaScript 中式无法自定义类型的。

  • Javascript 中的几个基本类型，都在对象类型中有一个”亲戚”：

    1. Number
    2. String
    3. Boolean
    4. Symbol

  • 注意：这里必须认识到 3 与 new Number(3)是完全不同的值， new 是对象类型，3 是一个数据类型

  • Number,String,和 boolean,三个构造器是两用的，当根 new 搭配使用时，他们产生的是对象,当直接调用时，他们表示强制类型转移
    var num = Number(“123”) //强制类型转移

  • Symbol 函数比较特殊，直接用 new 调用它会抛出错误，但它仍然是 Symbol 对象的构造器

关于 javaScript 的类型转换

  • JS 是弱类型的语言，所以类型转换发生的非常频繁
  • 在实践中推荐禁止==，要求程序员进行显示的类型转换后，用===比较。

详解 Number 和 String 之间的转换，对象跟基本类型之间的转换。

  • StringToNumber

    • 存在语法结构，支持转换的十进制，二进制，八进制和十六进制
    • parseInt 和 parseFloat 并不支持使用这个转换
    • 在任何环境下，都建议传入 parseInt 的第二个参数
    • parseFloat 则是直接把原字符串作为十进制来解析，他不会引入任何的其他进制。
    • 多数情况下,Number 是比 parseInt 和 parseFloat 更好的选择。

  • 装箱转换

    • 每一种基本数据类型 Number，String，Boolean，Symbol 在对应的类，所谓装箱转换，正是把基本数据类型转换为对应的对象，它是类型转换中一种相对重要的类。

    • 注意：全局的 Symbol 函数无法使用 new 来调用，但是我们仍可以利用装箱机制来的得到一个 Symbol 对象，我们可以利用一个函数的 call 方法来强迫产生装箱。

    • 我们可以定义一个函数，函数里面只有 return this，然后我们强调函数的 call 方法到一个 Symbol 类型的值上，这样就产生一个 SymbolObject.

    • 案例：我们可以用 console.log()看一下这个东西的 typeof，它的值是 object，我们使用 SymbolObject instanceof 可以看到，它是 Symbol 这个类的实例，我们找到它的 constructor 也是等于 Symbol 的，所以我们无论从那个角度看他都是 Symbol 装箱过的对象：

    • 注意：装箱机制会频繁产生临时对象，在一些对象性能要求较高的场景下，我们应该尽量避免对基本类型做装箱转换。
    • 使用内置的 Object 函数，我们可以在 javaScript 代码中显式调用装箱能力。
    • 每一类装箱对象皆有私有的 Class 属性，着些属性可以用 Object.prototype.toString 获取：

    • 重点注意的是，call 本身会产生装箱的操作，所以需要配合 typeof 来区分基本类型还是对象类型。

  • 拆箱转换

    • 在 javaScript 标准中，规定了 ToPrimitive 函数，它是对象类型到基本类型的转换(即,拆箱转换)。
    • 对象到 String 和 Number 的转换都遵循”先拆箱在转换”的规格。通过拆箱转换，把对象变成基本类型，在从基本类型转换为对应的 String 或者 Number
    • 拆箱转换会尝试调用 valueOf 和 toString 来获得拆箱后的基本类型。如果 valueOf 和 toString 都不存在，或者没有返回基本类型，则会产生类型错误 TypeError。

    • 上面最终会抛出 TypeError，这就说明了这个拆箱转换就失败了。

    • 到 String 的拆箱转换会优先调用 toString。

  • 在 ES6 之后，还允许对象通过显示指定 toPrimitive Symbol 来覆盖原有的行为

数据类型基础知识点补充

  • NaN 非数字

    1. 0/0 会得到非数字的情况
    2. NaN 不是独立的数据类型，而是一个特殊的数值，特殊的数值，它的数据类型依然属于 Number
    3. NaN 不等于任何值，包括他本身
    4. 在 Boolean 中 NaN 会被当作 false
    5. NaN 与任何数(包括它自己)的运算，得到的都是 NaN。
    6. parseInt()方法的第二个参数表示解析者的进制，默认的情况下参数为 10，即十进制
    7. isNaN()判断一个值是否为 NaN。(只对数值有效，如果传入的其他值，会被先转成数值，在判断)，Boolean 类型
    8. isNaN 对于空数组和只有一个数值成员的数组，isNaN 返回 false。
    9. isFinite 方法返回一个布尔值，表示某个值是否为正常的数值。(针对的 Infinity，-Infinity，NaN，undefined，null(true)，-1(true)等情况)

  • Base64 转码 Base64 是一种编码方式

    • 应用场景：

      1. 文本中包含一些不可打印的符号，比如 ASCII 码 0 到 31 的符号都无法打印出来，这时可以使用 Base64 编码，将它们转成可以打印的字符。
      2. 有时需要以文本格式传递二进制数据，那么也可以使用 Base64 编码。

    • 目的：不是为了加密，而是为了不出现特殊字符，简化程序的处理。

    • JavaScript 原生提供两个 Base64 相关的方法。

      1. btoa():任意值转为 Base64 相关的方法
      2. atob():Base64 编码转换为原来的值
      • 注意点：这两个方法不适合非 ASCLL 码的字符，会报错

      • 解决方案：
        要将非 ASCLL 码字符转为 Base64 编码，必须中间插入一个转码环节，在使用这两个方法

  • 对象 键值对

    1. 查看一个对象本身的所有属性，可以使用 Object.keys 方法。
    1. 属性的删除：delete 命令
    • delete 命令用于删除对象的属性，删除成功后返回 true
    • 注意，删除一个不存在的属性，delete 不报错，而返回一个 true
    • 如果对象中没有属性，但是执行 delete 命令照样会返回 true，因此，不能根据 delete 的命令结果，认定某个属性是存在的。
    • delete 命令返回 false 的情况，那就是该属性存在，且不得删除。

in 运算符

  • in 运算符用于检查对象是否包含某个属性（注意，检查的是键名，不是键值），如果包含就返回 true，否则返回 false。它的左边是一个字符串，表示属性名，右边是一个对象。

js 的函数对象的、apply()方法、arguments 属性，知识点补充

  • call()方法

    • call()方法是函数原型对象属性的方法，调用者为函数对象

    • 可以将一个对象指定为第一个参数，此时这个对象将会成为函数执行时的 this，从第二个参数起依次传入实参

    • 例：call(obj, a, b);第一个参数表示函数中的 this，a，b 表示实参

    • 在 fun 函数运行时指定的 this 值。需要注意的是，指定的 this 值并不一定是该函数执行时真正的 this 值，如果这个函数处于非严格模式下，则指定为 null 和 undefined 的 this 值会自动指向全局对象(浏览器中就是 window 对象)，同时值为原始值(数字，字符串，布尔值)的 this 会指向该原始值的自动包装对象。

    • 返回值：返回值是你调用的方法的返回值，若该方法没有返回值，则返回 undefined。

    • 可以让 call()中的对象调用当前对象所拥有的 function。你可以使用 call()来实现继承：写一个方法，然后让另外一个新的对象来继承它（而不是在新对象中再写一次这个方法）。

  • apply()方法

    • apply()方法是函数原型对象属性的方法，调用者为函数对象

    • 可以将一个对象指定为第一个参数，此时这个对象将会成为函数执行时的 this，实参需要封装到一个数组中，统一传递

    • 例：apply(obj, [a, b]);第一个参数表示函数中的 this，[a,b]数组表示实参数组

  • arguments 属性

    • arguments 属性是函数对象属性，在函数内调用

    • 在调用函数时，浏览器每次会传递两个隐含的参数：函数上下文对象 this、封装实参的对象 arguments；不管有没有定义形参，在调用函数时，所传的实参都会在 arguments 中保存

    • 可以通过 arguments[0]获得第一个实参，arguments[1]获得第二个实参…
