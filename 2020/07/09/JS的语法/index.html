脚本和模块

  • JavaScript 有两种源文件，
    • 一种叫做脚本：在 ES5 和之前的版本中，就只有一种源文件类型(就只有脚本)
    • 一种叫做模板：ES6 开始引入模块机制
  • 脚本是可以由浏览器或者 Node 环境引入执行的，而模板只能由 JavaScript 代码用 import 引入执行。
  • 从概念上，我们可以认为脚本具有主动性的 javaScript 代码字段，是控制宿主完成一定任务的代码；而模板是被动性的 JavaScript 代码段，是等待被调用的库。
  • 实际上模板和脚本之间的区别仅仅在于是否包含 import 和 export。
  • 脚本是一种兼容之前的版本的定义，在这个模式下，没有 import 就不需要处理加载”.js”文件问题。
  • 现代浏览器可以支持使用 script 标签引入模块或者脚本，如果引入模块，必须给 script 标签添加 type = “module”.如果引入脚本，则不需要 type。
  • 脚本中可以包含语句。模块中可以包含三种内容:import 声明，export 声明和语句。

import 声明和 export 声明

  • JavaScript 程序中包含：脚本(语句) 模块(import 声明，export 声明 语句)

import 声明

  • import 声明的两种用法
    1. 直接 import 引入一个模块
    2. 带 from 的 import ++》它能引入模块里的一些信息
    • 注意：直接 import 一个模块，只是保证了这个模块代码被执行,引用它的模块是无法获得任何信息的。
    • 带 from 的 import 意思就是引入模块中的一部分信息，可以把他们变成本地的变量。
  • 带 from 的 import 细节有细分由三种用法
    • import x from “./a.js”引入模块中导出的默认值。
    • import {a as x, modify} from “./a.js”;引入模块中的变量。
    • import * as x from “./a,js”把模块中所由的变量一类似对象属性的方式引入。
  • 第一种方式还可以跟后面两种组合使用。
    • import d,{a as x,modify} from “./a.js”
    • import d, * as x from “./a.js”
    • 注意语法要求不带 as 的默认值永远在最前。注意，这里的变量实际上仍然可以受到原来模板的控制。

  • 语句块的意义和好处在于：让我们可以把多行语句视为同一行语句，这样，if，for 等语句定义起来就比较简单了。不过，需要注意的是，语句块会产生作用域

空语句

  • 空语句就是一个独立的分号，实际上没有什么大用。
  • 空语句的存在仅仅是从语言的设计完备性的角度考虑,允许插入多个分号而不抛出错误。

if 语句

  • if 语句是条件语句
  • if 语句的作用是,在满足条件时执行它的内容语句，这个语句可以是一个语句块，这样就可以实现由条件的执行多个语句了。
  • if 语句还有 else 结构，用于不满足条件时执行，一种常见的用法是利用语句的嵌套能力，把 if 和 else 连写成多分支条件判断

switch 语句

  • switch 语句来实现分支，必须加上 break。

break 语句和 continue 语句

  • break 语句用于跳出循环语句或者 switch 语句,continue 语句用于结束本次循环并继续循环。
  • 这两个语句都是控制型语句
  • 带标签的 break 和 continue 可以控制自己被外层的那个语句结构消费，这样可以跳出复杂的语句结构。

with 语句

  • with 语句是一个非常巧妙的设计，但它把 javaScript
    的变量关系变得不可分析，所以一般都认为这种语句都属于糟粕。
  • with 语句把对象的属性在它内部的作用域内变成变量。

try 语句和 throw 语句

  • try 语句和 throw 语句用于处理异常。

  • 一般来说，throw 用于抛出异常，但是单纯从语言的角度，我们可以抛出任何值，也不一定是一场逻辑，但是为了保证语义清晰，不建议用 throw 表达任何非异常逻辑。

  • try 语句用于捕获异常，用 throw 抛出异常，可以在 try 语句的结构中被处理掉；try 部分用于表示捕获异常的代码段，catch 部分则捕获异常后做一些处理，而 finally 则是用于执行后做一些必须执行的清理工作。

  • catch 结构会创建一个局部的作用域，并且八一个变量写入其中，需要注意，在这个作用域，不能在声明变量 e 了，否则报错。

  • 在 catch 中重新抛出异常的情况非常常见。

  • finally 语句一般用于释放资源，它一定会被执行，即使在 try 中出现 return，finally 中的语句也一定要被执行。

debugger 语句

  • debugger 语句的作用是：通知调试器再次断点。在没有调试器挂载时，他不产生任何效果。

表达式语句

  • 事实上，真正能干活就只有表达式语句，其他的语句作用都是产生各种结构，来控制表达式语句执行，或者改变表达式语句的意义。

什么是表达式语句？？？？？？？

  • 表达式语句实际上就是一个表达式，它是由运算符来连接变量或者直接量构成的。
  • 一般来说,我们的表达式语句要么是函数调用，要么是赋值，要么是自增，自减，否则表达式计算的结果没有任何意义。

PrimaryExpression 主要表达式

  • 表达式的原子项：Primary Expression。它是表达式的最小单位，他所涉及的语法结构也是优先级最高的。
  • Proimary Expression 包含了各种”直接量”，直接量就是直接用某种语法写出来的具有特定类型的值。
  • Proimary Expression 还可以是 this 或者变量，在语法上，把变量称作”标识符应用”
  • 任何表达式加上圆括号，都被认为是 Primary Expression，这个机制使得圆括号成为改变运算优先顺序的手段。==》(a+b);
