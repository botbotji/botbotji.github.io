Completion 类型

  • Completion 类型：JavaScript 语句执行机制涉及的一种基础类型:Completion 类型。
  • 注意：function 这个是小写字母 输出：a 0
    所以这里 finally 执行了，return 语句也生效了，foo()返回了结果 0.

  • 如果在 finally 中加入 return 语句，会发生什么？？？？？(这是一个比较孤立的知识点)。
  • 输出结果：1
  • 分析：finally 中的 return “覆盖”了 try 中的 return。在一个函数中执行了两次 return，这已经超出了很多人的常识，也是其它语言中不会出现的一种行为。
  • 对于上面这种情况，背后有一套机制在运行（Completion Record）。

Completion Record 用与描述异常，跳出等语句执行过程。

  • Completion Record 表示一个语句执行完之后的结果，他有三个字段：
    • type 表示完成的类型， 有 break continue return throw 和 normal 几种类型;
    • value 表示语句的返回值，如果语句没有，则是 empty;
    • targrt 表示语句的目标，通常是一个 JavaScritpt 标签(标)
  • JavaScript 正是依靠语句的 Completion Record 类型，才可以在语句的复杂嵌套结构中，实现各种控制。

JavaScript 使用 Completion Record 类型，控制语句的执行过程

  • javaScript 中，我们把不带控制能力的语句称为普通语句。

    • 声明类语句：
      • var 声明
      • const 声明
      • let 声明
      • 函数声明
      • 类声明
    • 表达式语句
    • 空语句
    • debugger 语句

  • 这些语句在执行时，从前到后顺次执行(我们这里先忽略 var 和函数声明的预处理机制)，没有任何分支或者重复执行逻辑。

  • 普通语句执行后，会得到 type 为 normal 的 Completion Record，JavaScript 引擎遇到这样的 Completion Record，会继续执行下一条语句。

  • Chrome 控制台显示的正是语句的 Completion Record 的 value

语句块

  • 语句块就是拿大括号括起来的一组语句，它是一种语句的符合结构，可以嵌套
  • 语句块需要注意的是语句块内部的语句的 Comoletion Record 的 type 如果不为 normal，会打断语句块后续的语句执行。
  • 比如一个运行时的类型 type 为 return 的语句，出现在一个语句块中的情况。

控制型语句

  • 控制型语句带有 if，switch 关键字，它们会对不同的类型的 Conpletion Record 产生反应。
  • 控制类的语句分成两部分，一类是对其内部造成影响，如 if，switch，while/for，try。
  • 另一类是对外部造成影响如 break，continue，return，throw，这两类语句的配合，会产生控制代码执行顺序和执行逻辑的效果
  • 一般来说，for/while -break/continue 和 try -throw 这样的比较符合逻辑的组合，常见的
    但是，实际上，我们需要控制语句跟 break，continue，return，throw 四种类型与控制语句两两组合产生的效果。
